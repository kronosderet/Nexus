# v4.7.0-M1: Multi-source CC memory bridge

## Problem statement

`nexus_import_cc_memories` aktuálně skenuje jen jednu cestu — implicitně `~/.claude/projects/*/memory/*.md`. To pokrývá Claude Code dev sessions na **domácím PC**, ale zcela přehlíží Cowork sessions, které žijí v zcela jiné cestě:

| Stroj | CC dev session paměť | Cowork session paměť |
|---|---|---|
| Domácí PC (`krtek`) | `C:\Users\krtek\.claude\projects\<encoded>\memory\` | n/a |
| Školní notebook (`epic-bold-ptolemy` sandbox) | n/a | `/sessions/<sandbox-id>/mnt/.auto-memory/` |

Konkrétní symptom (zachycen 2026-05-02):

```
◈ Memory Bridge import — dry run
  9 memories scanned    ← všech 9 ze starých CC dev sessions
  · 0 new (would import)
  · 9 already on file (skipped)
```

Auto-memory **TUL projektu** (19 souborů, ~50 KB textu) — `MEMORY.md`, `project_kalg.md`, `project_kzed.md`, `persona_nalira.md`, `feedback_*.md`, `reference_*.md`, `user_profile.md` — **se nikdy do Ledgeru nedostane**, protože žije v sandbox cestě, kterou bridge nezná. Nalira tak mezi sessions ztrácí kontextovou kontinuitu, ledaže si všechno pamatuje sama z handoveru — což je křehké.

## Cíl

1. **Memory bridge umí skenovat víc cest najednou**, ne jen jednu pevnou.
2. **Cesty jsou konfigurovatelné** přes `nexus.config.json` (nebo CLI flag, nebo env var) — ne hardcoded v kódu.
3. **Dedup je per-content-hash, ne per-path** — pokud stejná memory existuje na obou strojích (např. `feedback_didactic_mode.md` se dělá v obou prostředích), bridge ji nezduplikuje, ale poznamenává její *všechny* zdrojové cesty.
4. **Backward compatible** — kdo nic nemění, dostane stejné chování jako v4.6.5.

### Po (návrh)

```typescript
// core/memory_bridge.ts
import { glob } from 'glob';
import { readFileSync } from 'fs';
import { createHash } from 'crypto';
import { getConfig } from './config';

interface MemorySource {
  name: string;
  path: string;          // glob pattern
  machineHint?: string;  // optional label, např. "domaci-pc"
  enabled: boolean;
}

interface MemoryEntry {
  contentHash: string;
  filePath: string;
  sourceName: string;
  machineHint?: string;
  frontmatter: Record<string, any>;
  body: string;
}

export async function importCcMemories(opts: {
  dryRun?: boolean;
  force?: boolean;
  project?: string;
  sourceFilter?: string;  // např. "cowork-sandbox" pro debug
}) {
  const config = getConfig();
  const sources = config.memoryBridge?.sources ?? [];

  if (sources.length === 0) {
    return { error: 'No memory sources configured', scanned: 0, imported: 0 };
  }

  const allEntries: MemoryEntry[] = [];
  const sourceErrors: Array<{ source: string; error: string }> = [];

  for (const source of sources) {
    if (!source.enabled) continue;
    if (opts.sourceFilter && source.name !== opts.sourceFilter) continue;

    try {
      const files = await glob(expandHome(source.path));
      for (const file of files) {
        const content = readFileSync(file, 'utf8');
        const { frontmatter, body } = parseFrontmatter(content);
        const contentHash = sha256(content);

        allEntries.push({
          contentHash,
          filePath: file,
          sourceName: source.name,
          machineHint: source.machineHint,
          frontmatter,
          body,
        });
      }
    } catch (err: any) {
      // Source unreachable (např. sandbox path neexistuje na domácím PC)
      // → log warning, NEselhej celý scan
      sourceErrors.push({ source: source.name, error: err.message });
    }
  }

  // Dedup
  const dedupStrategy = config.memoryBridge?.dedup?.strategy ?? 'path';
  const trackAll = config.memoryBridge?.dedup?.trackAllSources ?? false;
  const deduped = dedupEntries(allEntries, dedupStrategy, trackAll);

  // Insert do Ledgeru
  if (opts.dryRun) {
    return summarize(deduped, sourceErrors);
  }

  for (const entry of deduped) {
    await upsertAsDecision(entry, opts.force);
  }

  return summarize(deduped, sourceErrors);
}

function dedupEntries(
  entries: MemoryEntry[],
  strategy: 'path' | 'content-hash',
  trackAll: boolean
): MemoryEntry[] {
  const seen = new Map<string, MemoryEntry>();

  for (const entry of entries) {
    const key = strategy === 'content-hash' ? entry.contentHash : entry.filePath;

    if (!seen.has(key)) {
      seen.set(key, { ...entry, _allSources: trackAll ? [{ source: entry.sourceName, path: entry.filePath }] : undefined });
    } else if (trackAll) {
      const existing = seen.get(key)!;
      existing._allSources!.push({ source: entry.sourceName, path: entry.filePath });
    }
  }

  return Array.from(seen.values());
}

function expandHome(p: string): string {
  if (p.startsWith('~/')) {
    return p.replace('~', process.env.HOME ?? process.env.USERPROFILE ?? '');
  }
  return p;
}

function sha256(s: string): string {
  return createHash('sha256').update(s).digest('hex').slice(0, 16);
}
```

### Klíčové designové volby

1. **Per-source `try/catch`** — pokud sandbox path neexistuje (např. když běží Nexus na domácím PC, kde Cowork sandbox nemá), neshodí celý import. Jen warning v report.
2. **Content hash dedup** — pokud máš stejnou memory na dvou strojích (typicky persona, feedback), bridge ji vidí jako jednu, ale poznamenává obě cesty.
3. **`sourceFilter` opt** — pro debug, dá se zavolat jen jeden source. Užitečné při ladění.
4. **`expandHome` helper** — `~` musí fungovat na Windows (`USERPROFILE`) i POSIX (`HOME`).

## Migration

`v4.7.0-M1` migration:

```typescript
// migrations/v4.7.0-M1_memory_sources.ts
export const v4_7_0_M1 = {
  apply(store: NexusStore) {
    if (store.config.memoryBridge) return; // already migrated

    // Convert old single-path setup to new sources array
    const oldPath = store.config.legacyMemoryPath ?? '~/.claude/projects/*/memory/*.md';

    store.config.memoryBridge = {
      enabled: true,
      sources: [
        {
          name: 'cc-default',
          path: oldPath,
          enabled: true,
        },
      ],
      dedup: {
        strategy: 'path',           // backwards-compat strategy
        trackAllSources: false,
      },
    };

    delete store.config.legacyMemoryPath;
  },

  rollback(store: NexusStore) {
    if (!store.config.memoryBridge) return;
    const firstSource = store.config.memoryBridge.sources[0];
    store.config.legacyMemoryPath = firstSource?.path;
    delete store.config.memoryBridge;
  },
};
```

```typescript
describe('memory bridge — multi-source', () => {
  it('scans all configured sources', async () => {
    setConfig({ memoryBridge: { sources: [
      { name: 'a', path: 'tests/fixtures/memory_a/*.md', enabled: true },
      { name: 'b', path: 'tests/fixtures/memory_b/*.md', enabled: true },
    ], dedup: { strategy: 'path', trackAllSources: false }}});
    const result = await importCcMemories({ dryRun: true });
    expect(result.scanned).toBe(5); // 3 in a + 2 in b
  });

  it('dedupes by content hash when configured', async () => {
    // both sources have identical persona_nalira.md
    setConfig({ memoryBridge: { sources: [...], dedup: { strategy: 'content-hash', trackAllSources: true }}});
    const result = await importCcMemories({ dryRun: true });
    expect(result.scanned).toBe(5);
    expect(result.unique).toBe(4); // duplicate persona collapsed
    expect(result.entries.find(e => e.frontmatter.name === 'Nalira').sources).toHaveLength(2);

  it('continues on single-source failure', async () => {

    const result = await importCcMemories({ dryRun: true });
    expect(result.scanned).toBeGreaterThan(0);
    expect(result.sourceErrors).toHaveLength(1);
    expect(result.sourceErrors[0].source).toBe('broken');

  it('respects sourceFilter', async () => {
    setConfig({ memoryBridge: { sources: [
```

## Rollback

`v4.7.0-M1.rollback()` migration převede zpět na `legacyMemoryPath`. Bridge sám má fallback na default path když config sekce chybí. Tedy:

1. Spuštění Nexusu na v4.6.5 build s v4.7.0 store → config sekce `memoryBridge` bude ignorována, použije se default. Bez exception.
2. Žádná data v Ledgeru nepřijdou o sebe — decisions se nesmazávají.

## Open questions for Jakub

1. **Kde žije config?** Předpokládám `nexus.config.json` v `C:\Users\krtek\.nexus\`. Pokud máš oddělený soubor pro user-config (oddělený od stavu) — tam to patří. Pokud ne, vytvoř.
2. **Glob nebo regex?** Já jsem použila glob jako jednodušší. Pokud bys chtěl regex (silnější filter, např. `.*\.md$` ale ne `_test.md`), je to easy swap.
3. **Encoding cesty pro CC?** CC encoduje cestu k projektu (`-Users-krtek-Projects-xxx-`). Bridge by měl extrahovat skutečnou cestu pro `inferProjectName()`. Pokud to už dělá, zachovat. Pokud ne, přidat.
4. **Watcher mode?** Aktuální import je on-demand (uživatel zavolá `nexus_import_cc_memories`). Stálo by za to mít file watcher? Pravděpodobně overkill pro v4.7 — beam-down do v4.8.
5. **Sandbox path stability?** `/sessions/<sandbox-id>/mnt/.auto-memory/` má `<sandbox-id>` proměnnou (já jsem `epic-bold-ptolemy`, jiné session bude mít jiné jméno). Glob `/sessions/*/mnt/.auto-memory/*.md` by měl pokrýt všechny — ověř, že tvůj OS to umí. Na Windows přes WSL by mělo, ale chce to test.

## Changelog draft (release notes pro v4.7.0)

```markdown
## v4.7.0 — Multi-source CC memory bridge (2026-MM-DD)

### Added
- **Memory bridge now supports multiple sources.** Configure via `nexus.config.json` → `memoryBridge.sources[]`. Each source has its own path glob, optional machine hint, and enable flag.
- **Content-hash dedup.** Two sources can hold the same memory file (e.g. shared persona across machines) — bridge counts it once but tracks all source paths.
- **`nexus_import_cc_memories` accepts `source_filter`** for debugging.
- **CLI: `nexus memory sources`** — inspect current configuration.

### Changed
- `nexus_import_cc_memories` now reports `sourceErrors` array when individual sources fail (e.g. unreachable sandbox path on home machine). Previously the entire import would fail; now it's resilient.

### Migration
- Existing v4.6.x installations auto-migrate to single default source on first start. No action required.
- To add Cowork sandbox: edit `nexus.config.json` and add a source pointing to `/sessions/*/mnt/.auto-memory/*.md` (or wherever your Cowork instance writes auto-memory).

### Fixed
- N/A — pure feature add.
```
