## What the next session should know

### 1. The MCPB in Claude Desktop lags the repo

`nexus_version` reports whatever bundle was installed. After each release,
the user needs to reinstall the `.mcpb` (or reload the extension) to pick up
server/client code. Until then, MCP calls run the older adapter.

This is why the `v4.5.7-E1` edge migration had to be applied manually to
the live store in this session (the migration code lives in v4.5.7 store.ts
but the running MCPB is pre-v4.5.7). Pattern is:

```js
// node one-liner for any future "migration needs to fire against the live
// store without reinstalling MCPB" situation
node -e "
const fs = require('fs');
const d = JSON.parse(fs.readFileSync('C:/Users/kronos/.nexus/nexus.json','utf8'));
// ...mutate d...
d._appliedMigrations = d._appliedMigrations || {};
d._appliedMigrations['<migration-id>'] = new Date().toISOString();
fs.writeFileSync('C:/Users/kronos/.nexus/nexus.json', JSON.stringify(d, null, 2));
"
```

### 2. Shape-contract pattern for the standalone adapter

`server/mcp/localApi.ts` mimics the dashboard HTTP contract for MCP calls
running in standalone mode. A shape drift between dashboard and localApi
produced the v4.5.6 bug (silent "No results" for every `nexus_search` from
Claude Desktop). Regression guard: `tests/localApiSearch.test.ts` — extend
whenever a new endpoint gets both a dashboard route AND a localApi branch.

### 3. New reusable patterns from v4.5.x

- **Fuel correctness**: outlier bounds + confidence gating + inline sample size.
  Any UI statistic with `n < 3` should get the same treatment.
- **Inline-expand over modal** for drilldowns (Graph Overview Edge Types now,
  applicable to Fleet project cards / Centrality entries / Holes clusters next).
- **Proactive-signal cards on Command**: Overseer risks + Memory Bridge chip
  + new-events badge. Each has localStorage dismiss. Pattern applicable to
  any future "should the user know this right now?" signal.
- **Edge-migration with provenance**: orphan rel → canonical, append
  `[was rel=X]` to note. Idempotent via `_appliedMigrations`.

## Known issues + gotchas (fresh)

1. **Dashboard + MCPB version drift** — see section above. `api/status`
   vs `nexus_version` will disagree after a release until the extension
   reloads.

2. **Orphan decisions exist** (3 at time of this dock: #76, #157, #158) —
   decisions with zero graph edges. Not a bug; these are captured but
   never manually linked. The Holes view surfaces them. Auto-link
   (keyword + semantic) runs on decision creation, so these either
   matched nothing or pre-dated auto-link.

3. **Pre-existing `AIConnection` type drift** in `server/routes/overseer.ts`
   and `server/dashboard.ts` — ~9 errors carrying from earlier work.
   `npx tsc --noEmit -p server/tsconfig.json` makes them visible.

4. **Test worker flakes at high memory** — vitest has been ejecting
   workers under "System cannot execute" when RAM pressure is 97%+.
   Running individual test files works; full `npm run test` occasionally
   needs a retry. Not a code issue — OS-level pressure.
