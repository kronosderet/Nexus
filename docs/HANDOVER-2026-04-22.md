### 2. New CC-memory rule: sample nexus.json shape before writing

The v4.5.8 `/api/pulse/projects` 500 came from hand-constructing an activity
entry with `text:` instead of the canonical `message:`. Saved as feedback
memory `feedback_nexus_store_shape.md`. Rule: before any direct nexus.json
write, sample `Object.keys(d.<array>[0])` OR prefer `store.addActivity()` from
a script that imports `NexusStore`. This applies to ALL collections that
pulse.ts / dashboard / any route iterates with dotted field access.

### 3. Shape-drift pattern persists across client and localApi

v4.5.6 fix was `nexus_search` in localApi. v4.5.8 fix was `/memory` vs
`/cc-memory` in client `useApi.js`. Both were silent contract drifts between
a caller and the canonical endpoint. Every new client call site OR standalone
adapter branch should be paired with (a) the canonical endpoint in
`dashboard.ts`, (b) a test asserting the shape or route. Regression guard
precedent: `tests/localApiSearch.test.ts`.

### 4. Dashboard lags repo until restart

`npx tsx server/dashboard.ts` does not hot-reload. After any server-side
route change, `curl /api/status` still reports the old code until restart.
Client bundle IS served statically from `client/dist/`, so `npm run build`
+ browser hard-reload picks up client changes without dashboard restart.

### 5. MCPB lag still real

Installed MCPB in Claude Desktop lags repo until extension reload. Live-store
migrations stamped by running MCPB (pre-current-release) need manual
application pattern — see `HANDOVER-2026-04-20.md` for the Node one-liner.

## New patterns from v4.5.8

**Race-guarded `useEffect` fetch on selection change.** `VisualView` side panel
pattern: capture a `cancelled` flag in the effect, check before every
`setState`, clean up on teardown. Prevents stale responses overwriting fresh
state when user clicks through nodes quickly.

**Intra-cluster edge subset in holes response.** `/api/impact/holes` now emits
per-cluster `edges` (only edges where both endpoints are in the cluster). Pattern
reusable anywhere a parent response needs a "local subset of the full graph"
without the client re-computing.

**Deterministic circle layout for small graphs.** `ClusterMiniViz` uses
`theta = (i / size) * 2π - π/2` — stable across renders, no iteration cost,
readable for N ≤ 10. Prefer over force-directed for tiny SVGs.
