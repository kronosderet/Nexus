## New patterns from v4.4.4 (reuse when relevant)

**Week-over-week deltas in patterns.** `analyzeSessionPatterns` now splits
sessions into "last 7d" vs "prior 7d" and computes delta per metric. The
`DeltaBadge` component in Fuel.jsx takes `{delta, unit, lowerIsBetter, neutral}`
and color-codes accordingly. Pattern reusable for any time-series metric that
has a sensible direction.

**Server-computed projection overlay on calendar strip.** `weeklyProjection`
+ per-day `projectedWeekly`/`isProjectedEmpty` let the client render a tint
+ marker without computing burn rates client-side. Clean separation.

**Lifetime counters alongside active state.** Three-column row (Active · Ever
flagged · Resolved) gives the Conflicts tab a live state readout even at zero
active. Pattern applies anywhere a tab could feel "dead at zero" — could add
similar to Holes (current fragmented · ever detected · ever fixed).

**Per-type mute via compound chip.** Log TypeChips now pair a filter-to-type
button with a small eye icon for mute. Pattern: `<span><button>label</button>
<button>eye</button></span>` — the wrapping `span` carries state styling
(struck-through when muted). Good for any filter where "show me only this"
and "hide this from all" are both useful verbs.

**Q&A pair grouping for filtered chat history.** `qaPairs` is computed from
flat chatHistory by pairing each user turn with the following overseer turn.
Filter keeps both when either matches. Pattern for any paired-message UI.
