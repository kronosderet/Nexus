## Patterns learned + reused today

Copy these when relevant.

**Small-file batched commits.** When 3-5 small tasks live in the same
component, commit them together with per-task subsection in the body. Keeps
history grep-able. Example: `feat: Graph Centrality readability (#293 +
#294 + #295)` was one commit with three tasks.

**Hygiene migrations.** `_appliedMigrations` map on NexusData tracks applied
migration IDs. Pattern: `v4.x.y-H<N>` (H2 was casing, H3 was edge pruning).
Add as block in `store.ts::_runMigrations()`. Idempotent via `markApplied`.

**Two-phase dry-run for destructive actions.** `POST /ledger/auto-link?dry_run=true`
returns counts + samples without writes. UI shows preview, user confirms,
real call runs. Same pattern in `nexus_import_cc_memories`.

**Cross-tab navigation via root-level helpers.** `jumpToBlast(id)` /
`jumpToVisual(id)` in Graph root; passed as `onPickBlast` / `onPickVisual`
props to sub-views. Adding more crosslink triggers is one-line each.

**Shared combobox primitive.** `DecisionPicker.jsx` (added v4.4.1) ‚Äî
type-to-search across decision ID / text / project / tags. Used by Blast
Radius + Conflicts flag form. Ready for future orphan-linking + Overseer
"convert to decision" + any other "pick a decision" flow.

**WS_MAP slice invalidation.** `NexusProvider.jsx` defines which slices
refetch on which events. `task_update` / `session_created` / `activity`
now invalidate `fleet` + `pulse` alongside their primary targets (#247).

**Learned fuel rate: ~1.5% per small UI task.** Much lower than the stored
~5.9% avg which includes heavy feature work. Plan accordingly.

## Release ritual (established today)

1. Close tasks via `nexus_complete_task`
2. Log session via `nexus_log_session`
3. Version bump in 3 files (`package.json`, `cli/package.json`,
   `mcpb/manifest.json`)
4. `npm run test` ‚Äî must stay green
5. `npm run mcpb:pack` ‚Üí builds bundle
6. `npm run mcpb:test` ‚Üí verify tool count + version
7. Release commit with rich body
8. `git push origin main`
9. `git tag -a v<X> HEAD -m "..."` + `git push origin v<X>`
10. `gh release create v<X> --latest --title "..." --notes "..." <bundle-path>`
    - Stable release: `--latest`
    - Alpha/beta: `--prerelease`
11. Update `CHANGELOG.md` in the same release commit (learned in v4.4.3)

## Commit message style

Prefix: `feat` / `fix` / `refactor` / `chore` / `v<version>`. Short line
under 70 chars, body explains what/why (not what/how), references task IDs.
Multi-task commits list per-task subsection in body. Always ends with:

```
Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>
```
