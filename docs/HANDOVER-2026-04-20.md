## New patterns from v4.5.x (reuse when relevant)

**`<Chip>` primitive** — `client/src/components/Chip.jsx`. Shared pill for
any filter / scope / toggle UI. Two sizes (sm / md), states (active / muted /
default), renders as `<button>` when interactive or `<span>` when display-only.
Migrated call sites: Log time-range pills, Log sessions project chips,
Overseer Q&A date-range + project pills. Graph view tabs + Fuel PLAN_DETAILS
intentionally left as-is (different visual role).

**Overseer refine mode + transcript inheritance** — `server/routes/overseer.ts`
has two system prompts (`OVERSEER_SYSTEM` for analysis · `OVERSEER_REFINE_SYSTEM`
for follow-ups), a `formatHistory()` helper (caps at last 8 turns · 2000
chars per turn), and `buildSlimContext()` that drops per-project task dumps.
The client auto-switches to `refine` mode after the first exchange unless
the user manually picked a mode. Pattern applicable to any AI chat that
should skip strategic scaffolding for conversational follow-ups.

**Contradiction-scan pipeline** — two-stage engine in
`server/routes/overseer.ts`: `shortlistContradictionPairs()` uses cosine
similarity + lifecycle-divergence boost to prune O(n²) pairs, then
`buildContradictionPrompt()` packages the shortlist for the Overseer,
and `parseContradictionResponse()` tolerantly reads JSON (strips code
fences · handles prose prefixes · safe on malformed). Results stored as
`_suggestedContradictions` with sticky dismiss so the same pair never
re-surfaces. Pattern applicable to any future "scan the graph with an
LLM" work (e.g. deprecation candidates, orphan-cluster naming).

**Motion hooks + kinetic polish** — `client/src/hooks/useMotion.js`:
`useTweenedNumber` for RAF-driven numeric displays, `useWsFlash` for
"item arrived after mount" detection, `PREFERS_REDUCED_MOTION` for JS-side
bailouts. All CSS keyframes live at the bottom of `client/src/index.css`
with a single `@media (prefers-reduced-motion: reduce)` block disabling
everything in one place. Pattern to reuse: whenever adding a new animation,
extend the existing block rather than scattering reduced-motion checks
across files.

**`nexus_delete_task` + smoke-test self-cleaning** — the 27th MCP tool
closes a long-standing gap. `mcpb/smoke-test-bundle.mjs` now calls
`create → complete → delete` in sequence (exercises all three paths) and
post-processes `~/.nexus/nexus.json` to scrub residual activity + usage
entries after the MCP subprocess exits. Pattern to reuse: test fixtures
that write to production-shaped state should carry their own cleanup
— self-clean is better than leaving a README note ("delete this
manually later").

## Known issues + gotchas (updated)

1. **MCP in Claude Desktop runs in standalone mode** — Overseer
   async-start route (`/overseer/ask/start`) is unreachable without the
   dashboard HTTP server up. If you need to run an Overseer query from
   the MCP side, either (a) ensure `npm run dashboard` is running OR
   (b) use `curl`/`fetch` directly from a Node context.

2. **Rules of Hooks** — after the v4.5.0/v4.5.1 dance, remember that hook
   calls must precede ANY early-return. `useTweenedNumber` was placed
   below a `if (!data) return null` guard twice (ClockWidget, Fuel) and
   crashed both tabs. Rule of thumb: write the hooks first, then the
   guards. Tests mounted with data already present don't catch the
   null→populated transition where the bug surfaces — test-harness gap
   noted for future.

3. **Pre-existing `AIConnection` type drift** in `server/routes/overseer.ts`
   and `server/dashboard.ts` — ~9 errors that pre-date the v4.3 → v4.5
   work. Audit before patching unrelated files. `npx tsc --noEmit -p
   server/tsconfig.json` makes them visible.

4. **SessionStart hook runs from CC's CWD**, not the Nexus repo. When
   Claude is in a non-Nexus project, the hook's "Tests N/N green" grep
   won't find Nexus commit messages — that's correct behavior.

5. **Dashboard live version drift** — `curl /api/status` reports whatever
   version the dashboard PROCESS was started with, not what's on disk.
   The dashboard doesn't hot-reload on file change. If `api/status`
   reports an old version, restart the dashboard (`npm run dashboard`).

6. **`_suggestedContradictions` append-only** — dismissed suggestions
   are preserved in the store (needed for dedup so the same pair doesn't
   re-surface). Over time this could grow. No pruning yet. File as
   follow-up if scale becomes an issue.
